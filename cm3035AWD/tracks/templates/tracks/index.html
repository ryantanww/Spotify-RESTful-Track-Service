API List

1. GET - List of all tracks

Using the GET method, return a list of all the tracks in the database displaying only the id, track name and artists name.

Endpoint: http://127.0.0.1:8000/api/tracks/

2. POST - Add a new track

Using the POST method, add a new track (Validation via the serializers).

Endpoint: http://127.0.0.1:8000/api/tracks/create_track/

3. GET, PUT, DELETE - Track detail

Using the GET, PUT, and DELETE methods, return the track detail based on the track id, user will be able to update and delete the track on the same link (Validation via the serializers).

Endpoint: http://127.0.0.1:8000/api/tracks/{id}/

4. GET - Tracks by genre

Using the GET method, return a list of all the tracks filtered by the given genre.

Endpoint: http://127.0.0.1:8000/api/tracks/genre/{genre}/

5. GET - Popular tracks

Using the GET method, return a list of popular tracks with popularity sorted in descending order.

Default Endpoint: http://127.0.0.1:8000/api/tracks/popular_tracks/

Endpoint with parameter: http://127.0.0.1:8000/api/tracks/popular_tracks/?popularity=90

6. GET - High danceability tracks

Using the GET method, return a list of high danceable tracks with good tempo.

Default Endpoint: http://127.0.0.1:8000/api/tracks/high_danceability_tracks/

Endpoint with parameter: http://127.0.0.1:8000/api/tracks/high_danceability_tracks/?danceability=0.90&min_tempo=90&max_tempo=140
